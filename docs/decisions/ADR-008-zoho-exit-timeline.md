# ADR-008: Zoho Exit Timeline

**Status:** OPEN
**Date:** 2026-04-10
**Category:** Strategy
**Blocks:** Phase 1 execution planning

---

## Question

How aggressive should the Zoho migration be?

## Options

| Option | Pros | Cons |
|--------|------|------|
| **Parallel operation** (Zoho + ELIZA side by side for 6+ months) | Safe, can validate data integrity, easy rollback | Costly (Zoho subscription continues), dual entry burden |
| **Phased cutover** (module by module, 3-4 months) | Balanced risk, each module validated before next | Requires discipline to not get stuck in "parallel forever" |
| **Hard cutover** (big bang, 1 month) | Fast, forces adoption, stops Zoho cost immediately | High risk, no fallback, team resistance |
